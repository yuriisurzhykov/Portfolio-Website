# views/admin-login — AdminLoginPage

## 2026-07-19 — Фаза 4: `?from=` redirect + `router.refresh()` перед навигацией

**Что нужно сделать.** После логина администратор должен попасть туда,
куда изначально шёл (если его перекинул `proxy.ts` с `?from=/admin/work`),
а не всегда на дефолтную страницу.

**Как сделано.** `from` читается через `useSearchParams()` (поэтому
страница обёрнута в `<Suspense>` в `app/admin/login/page.tsx` — таково
требование Next.js для `useSearchParams()`), с проверкой
`from.startsWith("/admin")` — простая защита от open redirect (небезопасно
слепо доверять произвольному значению query-параметра, взятому из URL, как
адресу для редиректа).

`router.refresh()` вызывается ПЕРЕД `router.push(redirectTo)`, не после и
не вместо: Router Cache Next.js может отдать уже закэшированный,
"недо-аутентифицированный" рендер маршрута, если администратор уже
заходил на него в этой сессии до логина — `refresh()` форсирует реальный
повторный запрос с уже свежей cookie.

**Понятность для другого разработчика.** Комментарий на
`useRedirectTarget()` объясняет и источник `from` (`proxy.ts`), и почему
есть проверка префикса.

**SOLID.** Не применимо напрямую — простой компонент формы.
