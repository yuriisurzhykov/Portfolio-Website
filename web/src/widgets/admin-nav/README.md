# widgets/admin-nav — AdminNav

## 2026-07-19 — Фаза 4: шапка админки, и решение НЕ локализовать админку

**Что нужно сделать.** `/admin/*` нужна своя навигация (Journal/Work) и
способ выйти — но это не публичный сайт.

**Как сделано.** Своя, отдельная от публичного `<Nav/>` шапка — без
мобильного drawer'а, без переключателя языка/темы, без `useTranslation()`.
Осознанное решение: весь `web/src/views/admin-*` и `widgets/admin-nav`
пишут строки на английском напрямую в JSX, а не через `ln("...")` из
`shared/i18n`. Админку видит только владелец сайта — тот же человек,
который пишет код и ведёт эту самую документацию на русском в чате, но
интерфейс инструмента для СЕБЯ не обязан быть переведён на оба языка сайта
(в отличие от публичного контента, который читают посетители на двух
языках). Дублировать процесс "каждая строка — ключ в двух JSON-словарях"
(`shared/i18n/locales/{en,ru}.json`) для форм, которые видит один человек,
не даёт никакой ценности — YAGNI. Если позже появится второй администратор
с другим предпочитаемым языком интерфейса, это решение стоит пересмотреть
— пока это гипотетический сценарий, не реальное требование.

Кнопка "Log out" — `adminApi.logout()`, затем редирект на `/admin/login`
ВНЕ ЗАВИСИМОСТИ от того, успешен ли сам вызов: `proxy.ts` всё равно
перекинет обратно на `/admin/login`, как только access-token cookie
истечёт/пропадёт, так что неудачный запрос на logout не должен запирать
администратора на странице, которую он пытается покинуть.

**Понятность для другого разработчика.** Комментарий в коде прямо
объясняет решение не локализовывать — без него могло бы выглядеть как
недосмотр (забыли добавить `ln(...)`), а не осознанный выбор.

**Нагрузка.** Меньше, чем у публичного `<Nav/>` — нет i18n-контекста,
нет слушателя `hashchange`, нет мобильного меню.

**Расширяемость/заменяемость.** Если появится второй язык интерфейса
админки, i18n можно подключить сюда так же, как в публичном `<Nav/>` —
инфраструктура (`shared/i18n`) уже существует, просто не подключена здесь.

**Миграция/отказоустойчивость.** Не применимо.

**SOLID.** Single Responsibility: `AdminNav` отвечает только за навигацию
и выход, не за то, что происходит на самих страницах `/admin/journal`,
`/admin/work`.
